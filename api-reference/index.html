Compass API

API to interact with Compass. Include +access_token+ when authentication is required.

Activities /activities/

Programmatically interact with Activities on Compass. Typically only will read individual records from this resource.

List available activities

GET /activities

Example response
[
  {
    data: 'value'
    name: 'value'
    description: 'value'
  }
]

Create activity

POST /activities

Example response
{
  data: 'value'
  name: 'value'
  description: 'value'
}

Get activity

GET /activities/:id

Example response
{
  data: 'value'
  name: 'value'
  description: 'value'
}

Update given activity

PUT /activities/:id

Example response
{
  data: 'value'
  name: 'value'
  description: 'value'
}

Delete given Activity

GET /activities/:id

Example response
{
  data: 'value'
  name: 'value'
  description: 'value'
}

Activity sessions /activity_sessions/

Compass activity sessions are used to store progress information about a user's interaction with an activity. Session's can be created without an +access_token+ to allow anonymous access. If you have an ID of an anonymous session you should also be able to update it without an +access_token+.

List available activity sessions

GET /activity_sessions

Example response
[
  {
    percentage: 'value'
    time_spent: 'value'
    state: 'value'
    completed_at: 'value'
    data: 'value'
    temporary: 'value'
    activity_uid: 'value'
    anonymous: 'value'
  }
]

Create activity session

POST /activity_sessions
